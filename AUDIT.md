# 代码质量与规格完成性审计

审计日期：2026-07-23
审计对象：青龙重构版 `3.0.2-ql.2`、浏览器扩展 `2.0.0`、上游源码 `3.0.2`

## 结论

当前版本适合作为公开的 **Beta/实验性版本**，不应标记为生产稳定版。

基础架构、凭据隔离、青龙配置、主要 HTTP 任务、离线测试和许可证保留已经完成；但
Microsoft Rewards 使用未公开接口，且当前没有真实账号集成测试。必须依赖完整浏览器
DOM 的待领取积分、特殊 Punch Card 和部分 Quiz 页面交互没有做虚假兼容。

## 本轮修复

1. 修复 `dry-run` 仍刷新 OAuth 并写入状态文件的问题；
2. 补回原版的活动二次扫描；
3. PC 签到不再把无法解析或未确认的响应当成成功；
4. 活动校验令牌固定从 Rewards 域获取；
5. OAuth 优先使用轮换后的状态令牌，并在失败时回退到环境变量令牌；
6. 新增浏览器同步扩展，OAuth Token 仅会话存储并支持青龙 OpenAPI；
7. 增加 CI、许可证 NOTICE、安全说明和规格矩阵。

## 已知技术债

- HTTP 客户端没有通用自动重试，避免对非幂等领奖/签到请求造成重复提交；
- Cookie jar 是任务级实现，不覆盖 RFC 6265 的全部边界行为；
- 未公开 Microsoft API 可能在无通知情况下更改响应结构；
- 缺少使用专用测试账号的在线契约测试；
- 状态文件没有进程锁；同一账号不应并发执行多个青龙任务；
- 浏览器扩展当前以源码方式加载，尚未发布到 Chrome/Edge 扩展商店。

## 发布建议

- GitHub 首个版本标记为 `beta` 或 `experimental`；
- 不创建声称“防封号”的保证性描述；
- Release 中附上上游来源、SHA-256、许可证和已知限制；
- 浏览器扩展 OAuth 与青龙同步应先以 Beta 发布并进行人工授权测试。
